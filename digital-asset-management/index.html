The Tech Stack

React & Tailwind

Modern component architecture with fluid responsive states, lazy-loaded visual feeds, and async operation states.

FastAPI

Python web gateway serving high-throughput endpoints, utilizing asynchronous coroutines for heavy file handling.

PostgreSQL & pgvector

Relational storage with high-performance vector indexing. Handles standard photo metadata alongside 1536-dimensional image embeddings.

Google Gemini API

Powers multimodal zero-shot classification, tagging unstructured imagery and generating semantic search vector embeddings.

Key Features

Natural Language Search

Forget tags! Search photos like "workspace setup with coffee on desk" or "a dynamic sunset over the water." The semantic search uses vector cosine similarity to fetch high-confidence matches instantly.

Text-to-SQL Search

A secondary database-driven engine. Converts temporal and analytical text prompts (e.g. "photos taken last Wednesday") into valid PostgreSQL query syntax, resolving complex filtering logic on the fly.

Private & Local-First

Organizes media directly from your specified local directories. Keeps raw assets on your server, ensuring full ownership and data compliance for digital assets.

Auto-Tagging Pipeline

Automatically scans folders and runs zero-shot image classification. Discovers and tags objects, text, sentiments, dominant colors, and EXIF parameters without manual entry.

Performance Tuning

Utilizes TimescaleDB partitionings, pgvector's HNSW vector indexing, and caching of common queries to achieve sub-10ms response times for large visual directories.

Docker Orchestrated

Fully containerized system. Run a simple docker-compose up to spin up the React app, FastAPI server, and pre-configured pgvector database container.

System Architecture

The platform splits operations into two optimized pipelines: the Ingestion Pipeline (parsing folders and generating embeddings) and the Query Pipeline (handling real-time search queries).

Engineering Challenges

01 Optimizing Vector Search Latency

Problem: Standard linear scan (flat) vector lookups scale linearly with the gallery size, creating high latency spikes when searching thousands of photos.

Solution: We implemented the Hierarchical Navigable Small World (HNSW) index using pgvector. This constructs a multi-layered graph for proximity matching, reducing search complexity to logarithmic time.

-- Create HNSW index on the embeddings field for cosine similarity
CREATE INDEX ON photos USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);

02 Securing Natural Language to SQL

Problem: Using LLMs to construct SQL statements from free-form user text is a high-risk vector for SQL injection if the model decides to delete or alter records.

Solution: Implemented a multi-tier safety bridge. First, SQL generation queries are constrained by system prompt rules to read-only schemas. Second, the backend parses the generated abstract syntax tree (AST) via Python's SQL parsing library to verify only SELECT operations are present before executing in database transactions.
